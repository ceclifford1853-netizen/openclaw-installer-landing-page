Node.js Version Troubleshooting

Resolve Node.js and npm Issues with OpenClaw

Node.js 22 LTS Overview

The OpenClaw One-Click Installer uses Node.js 22 LTS (Long-Term Support), the latest stable version optimized for production environments. This version includes performance improvements, security patches, and enhanced compatibility with modern JavaScript features.

✓ Node.js 22 LTS is automatically installed and pinned by the OpenClaw installer. No manual version management is required.

Checking Your Node.js Version

Verify that Node.js 22 is installed correctly:

Check Node.js Version

node --version

Expected output: v22.x.x (where x represents the minor and patch versions)

Check npm Version

npm --version

Expected output: 10.x.x or higher

View Detailed Node.js Information

node -v && npm -v && npm list -g openclaw

Common Node.js Issues and Solutions

Issue: "node: command not found"

Cause: Node.js is not installed or not in your system PATH.

Solution:

  • Run the OpenClaw One-Click Installer again to reinstall Node.js
  • Or manually install Node.js 22 LTS from https://nodejs.org
  • Restart PowerShell after installation to refresh the PATH

Issue: "npm: command not found"

Cause: npm is not properly installed with Node.js.

Solution:

  • Verify Node.js installation: node --version
  • Reinstall Node.js using the Windows installer from nodejs.org
  • Ensure you have Administrator privileges during installation

Issue: "npm ERR! code EACCES"

Cause: Permission issues with npm global packages.

Solution:

  • Run PowerShell as Administrator
  • Clear npm cache: npm cache clean --force
  • Reinstall OpenClaw: npm install -g openclaw@latest

Issue: "Wrong version of Node.js installed"

Cause: An older or newer version of Node.js is installed.

Solution:

  • Uninstall the current Node.js version via Control Panel → Programs → Programs and Features
  • Download and install Node.js 22 LTS from nodejs.org
  • Verify installation: node --version

Issue: "OpenClaw requires Node.js 22"

Cause: You're running an incompatible Node.js version.

Solution:

# Check current version
node --version

# If not v22.x.x, upgrade to Node.js 22 LTS
# Download from https://nodejs.org/en/download/
# Or use a Node.js version manager like nvm-windows

Managing Multiple Node.js Versions

If you need to work with multiple Node.js versions, consider using nvm-windows (Node Version Manager for Windows):

Install nvm-windows

# Download from https://github.com/coreybutler/nvm-windows/releases
# Run the installer and follow the setup wizard

Install Node.js 22 via nvm

nvm install 22
nvm use 22

List Installed Versions

nvm list
Important: If you use nvm-windows, ensure you're using Node.js 22 when running OpenClaw. Use nvm use 22 before starting the daemon.

npm Global Package Management

View Global npm Packages

npm list -g --depth=0

Update npm Itself

npm install -g npm@latest

Verify OpenClaw Installation

npm list -g openclaw

Reinstall OpenClaw

npm uninstall -g openclaw
npm install -g openclaw@latest

Refreshing Your PATH Environment Variable

After installing Node.js, your system PATH must be updated to recognize the node and npm commands:

Automatic (Recommended)

The OpenClaw One-Click Installer automatically refreshes your PATH. Simply close and reopen PowerShell.

Manual Refresh in PowerShell

$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")

Verify PATH is Updated

$env:Path -split ";" | Select-String "nodejs"

Performance Optimization

Enable Node.js Experimental Features (Optional)

Some OpenClaw features may benefit from experimental Node.js features:

node --experimental-modules app.js

Monitor Node.js Memory Usage

node --max-old-space-size=4096 app.js

Enable Node.js Profiling

node --prof app.js

Frequently Asked Questions

Can I use Node.js 20 or 21 instead of 22?

OpenClaw is optimized for Node.js 22 LTS. While older versions may work, we recommend using 22 for best performance and security.

Is Node.js 22 stable for production?

Yes. Node.js 22 is an LTS (Long-Term Support) version, meaning it receives security updates and bug fixes for an extended period.

How do I update Node.js to a newer version?

Download the latest Node.js 22 LTS installer from nodejs.org and run it. It will upgrade your existing installation.

What if the installer fails to install Node.js?

Check your internet connection, ensure you have Administrator privileges, and try installing Node.js manually from nodejs.org. Then run the OpenClaw installer again.
