ClawJacked Prevention Guide

Secure Your OpenClaw Installation Against External Threats

What is ClawJacked?

ClawJacked is a theoretical attack vector where a malicious website or external actor could attempt to hijack your OpenClaw AI daemon through unauthorized WebSocket connections or remote access exploits. The attack targets the gateway port (default 3000) if it's exposed to the internet or accessible from untrusted networks.

Important: By default, the OpenClaw One-Click Installer implements ClawJacked Shield, which binds the gateway to 127.0.0.1 (localhost only). This prevents external access automatically.

How ClawJacked Shield Works

ClawJacked Shield is a multi-layered security approach that protects your OpenClaw installation:

1. Local-Only Binding

The gateway listens exclusively on 127.0.0.1:3000, making it inaccessible from external networks. Only applications running on your local machine can communicate with OpenClaw.

2. Windows Firewall Integration

Firewall rules are automatically configured to allow only local traffic on port 3000. Remote connections are blocked at the OS level.

3. 2026 Security Patches

The installer applies the latest security patches and hardening configurations automatically during installation.

4. Systemd Daemon Protection (x64)

On 64-bit systems with WSL2, the OpenClaw daemon runs under systemd with restricted permissions, limiting the attack surface.

Verification Steps

After installation, verify that ClawJacked Shield is active:

Check Gateway Address

openclaw config get gateway.address

Expected output: 127.0.0.1

Verify Firewall Rules

netsh advfirewall firewall show rule name="OpenClaw Local"

This should display the firewall rule restricting access to localhost.

Test Local Access

curl http://localhost:3000

This should succeed, confirming local access is working.

Test External Access (Should Fail)

curl http://YOUR_IP_ADDRESS:3000

This should fail or timeout, confirming external access is blocked.

Best Practices for Maximum Security

1. Never Expose the Gateway Externally

Do not modify the gateway address to 0.0.0.0 or your machine's IP address unless you have a specific reason and understand the security implications.

2. Use a Reverse Proxy for Remote Access

If you need remote access to OpenClaw, use a secure reverse proxy (like Nginx with SSL/TLS) on a different port, not the direct gateway port.

3. Keep Windows Updated

Regularly update Windows and install security patches. This ensures your firewall and OS-level protections remain current.

4. Monitor Firewall Logs

Periodically review Windows Firewall logs to detect any suspicious connection attempts:

Get-WinEvent -LogName Security | Where-Object {$_.ID -eq 5152}

5. Secure Your API Keys

Store OpenClaw API keys and credentials in environment variables or secure vaults, never in code or configuration files.

6. Run Regular Security Audits

Use the built-in health check to verify your installation:

openclaw doctor

7. Disable Unnecessary Services

Only enable OpenClaw services you actually use. Disable or remove unused features to reduce the attack surface.

Responding to Security Incidents

If you suspect a security breach or unauthorized access:

  • Stop the daemon immediately: openclaw stop
  • Review logs: openclaw logs
  • Check firewall rules: netsh advfirewall firewall show rule all
  • Reinstall if necessary: Run the One-Click Installer again to reset security configurations
  • Report security issues: Contact the OpenClaw team through the official channels

Frequently Asked Questions

Can I access OpenClaw from another computer on my network?

By default, no. The gateway is bound to localhost only. If you need network access, use a reverse proxy with proper authentication and encryption.

What if I accidentally expose the gateway?

Immediately run the installer again to reset the security configuration. The ClawJacked Shield will be reapplied.

Is ClawJacked Shield active on 32-bit systems?

Yes. The local-only binding and firewall rules are applied to both 32-bit and 64-bit systems.

Can I disable ClawJacked Shield?

You can modify the gateway address, but this is not recommended. If you do, you assume full responsibility for securing the exposed port.
