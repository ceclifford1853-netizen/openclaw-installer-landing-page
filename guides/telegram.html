Telegram Bot Setup Guide

Enable OpenClaw Notifications and Remote Management via Telegram

What is the OpenClaw Telegram Bot?

The OpenClaw Telegram bot allows you to receive real-time notifications about your OpenClaw daemon status, manage configurations, and access remote commands through Telegram. This guide walks you through the setup process after your initial OpenClaw installation.

Note: Telegram bot integration is optional. Your OpenClaw installation works perfectly without it, but the bot provides convenient remote access and notifications.

Prerequisites

Before setting up the Telegram bot, you need:

  • A Telegram account (download from https://telegram.org)
  • OpenClaw already installed and running on your Windows system
  • Administrator access to your Windows machine
  • Your Telegram user ID (we'll show you how to find this)

Step 1: Create a Telegram Bot

1.1 Open Telegram and Find BotFather

Open Telegram and search for @BotFather, the official Telegram bot for creating new bots.

1.2 Start BotFather

Click "Start" to begin the conversation with BotFather.

1.3 Create a New Bot

Send the command /newbot to BotFather.

BotFather will ask for:

  • Bot Name: e.g., "OpenClaw Monitor"
  • Bot Username: e.g., "openclaw_monitor_bot" (must end with "_bot")

1.4 Copy Your Bot Token

BotFather will provide a bot token that looks like:

123456789:ABCDefGhIjKlMnOpQrStUvWxYz

Save this token securely. You'll need it in the next step.

Step 2: Find Your Telegram User ID

2.1 Search for IDBot

In Telegram, search for @userinfobot or @getidsbot.

2.2 Get Your User ID

Click "Start" and the bot will immediately reply with your user ID, which looks like:

Your user ID: 123456789

Save this ID. You'll need it to authorize the OpenClaw bot.

Step 3: Configure OpenClaw with Your Bot Token

3.1 Open PowerShell as Administrator

Right-click PowerShell and select "Run as Administrator".

3.2 Set Your Bot Token

openclaw config set telegram.bot_token YOUR_BOT_TOKEN_HERE

Replace YOUR_BOT_TOKEN_HERE with the token from BotFather.

3.3 Authorize Your Telegram User ID

openclaw config set telegram.user_id YOUR_USER_ID_HERE

Replace YOUR_USER_ID_HERE with your user ID from the IDBot.

3.4 Enable Telegram Notifications

openclaw config set telegram.enabled true

3.5 Restart OpenClaw Daemon

openclaw restart

Step 4: Test Your Bot Connection

4.1 Send a Test Message

In Telegram, find your OpenClaw bot (the one you created with BotFather) and click "Start".

4.2 Verify Connection

The bot should respond with a welcome message confirming the connection is active.

4.3 Check OpenClaw Logs

openclaw logs

You should see entries confirming the Telegram bot is connected.

Available Bot Commands

Status Commands

  • /status - Check OpenClaw daemon status
  • /health - Run health check
  • /logs - View recent logs
  • /version - Check OpenClaw version

Control Commands

  • /start - Start the daemon
  • /stop - Stop the daemon
  • /restart - Restart the daemon
  • /config - View current configuration

Information Commands

  • /help - Display available commands
  • /about - About OpenClaw

Security Best Practices

Important Security Notes:
  • Never share your bot token with anyone. It grants full control of your bot.
  • Only authorize your own Telegram user ID. The bot will only respond to authorized users.
  • Use strong Telegram account security. Enable two-factor authentication on your Telegram account.
  • Monitor bot activity. Regularly check your OpenClaw logs for unexpected commands.
  • Disable the bot if not in use. Set telegram.enabled false to disable notifications.

Troubleshooting

Bot Not Responding

Solution:

  • Verify your bot token is correct: openclaw config get telegram.bot_token
  • Ensure Telegram notifications are enabled: openclaw config get telegram.enabled
  • Restart the daemon: openclaw restart
  • Check logs: openclaw logs

Unauthorized User Error

Solution:

  • Verify your user ID is correct: openclaw config get telegram.user_id
  • Make sure you're using the exact user ID from the IDBot (no spaces or extra characters)
  • Restart the daemon: openclaw restart

Commands Not Working

Solution:

  • Ensure OpenClaw daemon is running: openclaw status
  • Check that your Telegram user is authorized
  • Verify the command syntax is correct (commands are case-sensitive)
  • Review logs for error messages: openclaw logs

Disabling the Telegram Bot

If you want to disable Telegram notifications:

openclaw config set telegram.enabled false
openclaw restart

To remove the bot completely:

openclaw config unset telegram.bot_token
openclaw config unset telegram.user_id
openclaw config unset telegram.enabled
openclaw restart

Frequently Asked Questions

Is Telegram bot integration secure?

Yes. Only authorized user IDs can control the bot, and all communication is encrypted through Telegram's secure servers.

Can multiple users control the same bot?

The basic setup authorizes one user ID. For multiple users, you'll need to configure additional authorized IDs in your OpenClaw settings.

What if I lose my bot token?

You can create a new bot with BotFather and update your OpenClaw configuration with the new token.

Does the bot work if OpenClaw is offline?

No. The bot can only communicate with OpenClaw when the daemon is running. It will notify you if the daemon goes offline.
